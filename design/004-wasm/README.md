# WASM Build of ox-jsdoc for Browser

## Context

ox-jsdoc's `parse_comment` should run in the browser via WebAssembly.
Currently only a NAPI binding (Node.js only) exists. A standalone WASM crate
using `wasm-pack` + `wasm-bindgen` provides the simplest browser-compatible
path.

## Approach: wasm-pack + wasm-bindgen

Chosen over napi-rs WASM target (`wasm32-wasip1-threads`) because:

- `parse_comment` is synchronous, single-threaded — no WASI threading needed
- Output is a JSON string — `wasm-bindgen` `String` is sufficient
- No `SharedArrayBuffer` / cross-origin isolation headers required
- No `@napi-rs/wasm-runtime` — smaller bundle size
- `wasm-bindgen` is mature and stable

## Rust WASM Binding

```rust
use wasm_bindgen::prelude::*;
use oxc_allocator::Allocator;
use ox_jsdoc::{ParseOptions, parse_comment, serialize_comment_json};

#[wasm_bindgen]
pub fn parse(source_text: &str, fence_aware: Option<bool>) -> JsValue {
    let allocator = Allocator::default();
    let options = ParseOptions {
        fence_aware: fence_aware.unwrap_or(true),
        inline_code_aware: false,
    };
    let output = parse_comment(&allocator, source_text, 0, options);

    let ast_json = match output.comment {
        Some(ref comment) => serialize_comment_json(comment, None, None),
        None => "null".to_string(),
    };

    // Build { astJson, diagnostics } via js_sys
    let result = js_sys::Object::new();
    js_sys::Reflect::set(&result, &"astJson".into(), &ast_json.into()).unwrap();
    let diag_array = js_sys::Array::new();
    for d in &output.diagnostics {
        let obj = js_sys::Object::new();
        js_sys::Reflect::set(&obj, &"message".into(), &d.to_string().into()).unwrap();
        diag_array.push(&obj);
    }
    js_sys::Reflect::set(&result, &"diagnostics".into(), &diag_array.into()).unwrap();
    result.into()
}
```

No lifetime issue — `serialize_comment_json` returns an owned `String` and
arena references stay within the function scope (same pattern as the napi
binding).

## JS Wrapper

```js
import init, { parse as parseWasm } from './pkg/ox_jsdoc_wasm.js'

let initialized = false

export async function initWasm(wasmUrl) {
  if (!initialized) {
    await init(wasmUrl)
    initialized = true
  }
}

export function parse(sourceText, options) {
  if (!initialized) throw new Error('Call initWasm() before parse()')
  const result = parseWasm(sourceText, options?.fenceAware)
  return {
    get ast() {
      const value = result.astJson === 'null' ? null : JSON.parse(result.astJson)
      Object.defineProperty(this, 'ast', { value })
      return value
    },
    diagnostics: result.diagnostics
  }
}
```

Same `{ ast, diagnostics }` shape as the napi binding's `parse()`.

### What `initWasm()` does

`initWasm()` calls the `init()` function auto-generated by wasm-bindgen,
which performs the following steps:

1. **Fetch the `.wasm` binary** — downloads the WebAssembly file over the
   network (or from a bundler asset)
2. **Compile** — calls `WebAssembly.instantiateStreaming()` to compile the
   binary into a WASM module
3. **Initialize linear memory** — allocates and initializes the WASM linear
   memory region
4. **Connect JS glue code** — binds the import/export bridge that
   wasm-bindgen generates (string marshalling, memory management helpers, etc.)

Browsers require `fetch` + `WebAssembly.instantiateStreaming` which are
asynchronous, so `initWasm()` must complete before `parse()` can be called.
The optional `wasmUrl` argument allows callers to specify a custom URL for the
`.wasm` file; when omitted, the default path generated by wasm-pack is used.

## Implementation Steps

1. Add `wasm32-unknown-unknown` target to Rust toolchain
2. Create `wasm/ox-jsdoc/` with `Cargo.toml`, `src/lib.rs`
3. Update workspace `Cargo.toml` to add `wasm/*` to members
4. Update `pnpm-workspace.yaml` to add `wasm/*`
5. Compile check: `cargo build -p ox_jsdoc_wasm --target wasm32-unknown-unknown`
6. Create `package.json`
7. Build: `cd wasm/ox-jsdoc && pnpm run build`
8. Create `src-js/index.js` and `src-js/index.d.ts`
9. Add vitest tests in `wasm/ox-jsdoc/test/parse.test.ts`
10. Run tests

## Testing

WASM binding tests use vitest with browser environment.
Test files are placed under `wasm/ox-jsdoc/test/`.

### What to test

- `initWasm()` completes without error
- `parse()` throws before `initWasm()` is called
- `parse()` converts a well-formed JSDoc block into an AST
- The return value has `{ ast, diagnostics }` shape
- `ast.type` equals `"JsdocBlock"`
- Tag fields `tag`, `rawType`, `name`, `description` are correctly extracted
- Malformed input produces `diagnostics`
- Non-JSDoc input returns `ast: null` with diagnostics
- `fenceAware` option works
- Optional parameter syntax `[name=default]` is parsed correctly

### Test file

````ts
// wasm/ox-jsdoc/test/parse.test.ts
import { describe, expect, it, beforeAll } from 'vite-plus/test'
import { initWasm, parse } from '../src-js/index.js'

beforeAll(async () => {
  await initWasm()
})

describe('parse', () => {
  it('parses a basic param tag', () => {
    const result = parse('/** @param {string} id - The user ID */')
    expect(result.diagnostics).toEqual([])
    expect(result.ast).not.toBeNull()
    expect(result.ast!.type).toBe('JsdocBlock')
    expect(result.ast!.tags[0].tag).toBe('param')
    expect(result.ast!.tags[0].rawType).toBe('string')
    expect(result.ast!.tags[0].name).toBe('id')
    expect(result.ast!.tags[0].description).toBe('The user ID')
  })

  it('parses description with inline tags', () => {
    const result = parse('/** See {@link Foo} for details. */')
    expect(result.diagnostics).toEqual([])
    expect(result.ast!.description).toContain('{@link Foo}')
    expect(result.ast!.inlineTags.length).toBe(1)
    expect(result.ast!.inlineTags[0].tag).toBe('link')
  })

  it('returns diagnostics for malformed input', () => {
    const result = parse('/** {@link Foo */')
    expect(result.diagnostics.length).toBeGreaterThan(0)
  })

  it('rejects non-jsdoc input', () => {
    const result = parse('/* plain */')
    expect(result.ast).toBeNull()
    expect(result.diagnostics.length).toBeGreaterThan(0)
  })

  it('supports fenceAware option', () => {
    const source = '/**\n * @example\n * ```ts\n * @decorator()\n * ```\n * @returns {void}\n */'
    const result = parse(source, { fenceAware: true })
    expect(result.ast!.tags.length).toBe(2)
  })

  it('handles optional parameter syntax', () => {
    const result = parse('/** @param {string} [name=default] - desc */')
    expect(result.ast!.tags[0].optional).toBe(true)
    expect(result.ast!.tags[0].defaultValue).toBe('default')
  })

  it('handles empty comment', () => {
    const result = parse('/** */')
    expect(result.diagnostics).toEqual([])
    expect(result.ast).not.toBeNull()
    expect(result.ast!.tags.length).toBe(0)
  })
})

describe('initWasm', () => {
  it('can be called multiple times safely', async () => {
    await initWasm()
    await initWasm()
    const result = parse('/** @param x */')
    expect(result.ast).not.toBeNull()
  })
})
````

## Potential Issues

- **oxc_diagnostics on WASM**: `miette` may pull in platform-specific
  features. Mitigation: the binding only calls `.to_string()` on diagnostics.
  If compilation fails, gate behind `cfg(target_arch = "wasm32")` or convert
  to strings in the core crate.
- **WASM binary size**: `serde_json` formatting can bloat output. Expected
  ~200-400 KB. Optimize with `wasm-snip` if needed.
